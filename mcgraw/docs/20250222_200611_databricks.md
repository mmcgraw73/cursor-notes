# databricks notes

### Data Lakes

- Data Lakes are a way to store all data in its raw form, without structure.
- This is different from traditional data warehouses, which are designed to store structured data.
- Data Lakes are often used for big data analytics, as they can handle the large volumes of data that are generated by modern applications.

### SQL specific notes

- data imputation: the process of filling in missing values in a dataset
- data transformation: the process of changing the format or structure of a dataset
- data enrichment: the process of adding additional information to a dataset
- data validation: the process of checking the accuracy and consistency of a dataset
- data profiling: the process of creating a profile of a dataset
- data masking: the process of hiding sensitive information in a dataset

best practice for imputed data:

- use the median for numeric data
- use the mode for categorical data
- use the most recent value for time series data
- if possible add a new column that indicates imputed data

schema general definition:

- a schema is a collection of tables
- a table is a collection of rows

schema best practices:

- should be lowercase

### when to choose between Standard (Multi-node) and Single-node clusters in Databricks:

Standard (Multi-node) Clusters Preferred When:
Data Processing Needs:
Large datasets (TB+)
Complex computations
Parallel processing required
Production workloads
High-performance needs

Single-node Clusters Preferred When:
Development/Testing:
Prototyping code
Testing workflows
Learning/training
Small datasets
Quick experiments

Cost Considerations:
Budget constraints
Development environment
Proof of concept
Personal projects
Learning exercises

Simple Workloads:
Small data analysis
Code development
Light data exploration
Documentation
Testing

Rule of thumb: Use single-node for development/testing
and standard clusters for production/large-scale processing.

### Feature Store Concepts

- Feature Store is a feature store that is used to store features for machine learning
- Features are a type of data that is used to train machine learning models
- Features are a type of data that is used to make predictions
- Features are a type of data that is used to evaluate machine learning models
- Features are a type of data that is used to deploy machine learning models
- manages features, or input data to a machine learning model
